Basics

📝 How to use

```mermaid
flowchart LR
  A --> B
```
  • Put code in a mermaid fenced block in Markdown — it renders as SVG.
  • First line = diagram type; direction: TD/TB (top-down), LR (left-right), RL, BT.
  • Version-controlled, diff-able, editable — no binary image files.
Flowchart

🔷 Nodes & edges

flowchart LR
  A[Rectangle]
  B(Rounded)
  C{Decision}
  D((Circle))
  E[(Database)]
  A --> B            %% arrow
  A --- C            %% line, no arrow
  A -->|label| B     %% labelled edge
  C -->|yes| D
  C -->|no| E
  A -.-> B           %% dotted
  A ==> B            %% thick

📦 Subgraphs (a pipeline)

flowchart LR
  subgraph Ingest
    S[(Source)] --> K[Kafka]
  end
  subgraph Process
    K --> T[Transform]
  end
  T --> W[(Warehouse)]
  W --> BI[Dashboard]

Group related nodes with subgraph … end to show zones/stages.

🎨 Notes & styling

flowchart LR
  A[OK] --> B[Warn]
  classDef bad fill:#3d1f24,stroke:#ff6b6b;
  class B bad
  %% comments start with %%
  click A "https://…" "tooltip"

Keep diagrams small & focused; one idea per diagram. Split large systems into several.

💡 Tips

  • Prototype live at mermaid.live, then paste into your doc/repo.
  • Pick a direction (LR for pipelines, TD for hierarchies) and stay consistent.
  • Label edges (batch/stream, data type); use [(cylinder)] for data stores.
  • Renders in GitHub/GitLab READMEs, Notion, Obsidian, and this app's lessons.
