Arrow basics

🧠 What Arrow is

  • A language-agnostic columnar in-memory format — the standard many tools share.
  • Lets pandas, Polars, DuckDB, Spark exchange data zero-copy (no serialize/parse).
  • Parquet = the on-disk columnar format; Arrow is its in-memory counterpart.

🧱 Arrays & tables

import pyarrow as pa

arr = pa.array([1, 2, 3], type=pa.int64())
tbl = pa.table({"id": [1,2], "name": ["a","b"]})
tbl.schema        tbl.num_rows    tbl.column_names
tbl.column("id")   tbl.slice(0, 100)
tbl.select(["id"])   tbl.filter(pa.array([True, False]))
tbl.combine_chunks()

🏷️ Schema & types

schema = pa.schema([
    ("id", pa.int64()),
    ("ts", pa.timestamp("us", tz="UTC")),
    ("tags", pa.list_(pa.string())),
    ("cust", pa.struct([("id", pa.int64())])),
])
tbl.cast(schema)
# int8/16/32/64, float32/64, string, bool,
#   date32, timestamp, decimal128, list_, struct, dictionary

dictionary type = Arrow's categorical (dedups repeated strings → big memory win).

Parquet IO

💾 Read & write Parquet

import pyarrow.parquet as pq

tbl = pq.read_table("f.parquet")
tbl = pq.read_table("f.parquet",
        columns=["id", "amt"],           # projection
        filters=[("dt", "=", "2026-01-01")])  # predicate pushdown

pq.write_table(tbl, "out.parquet",
        compression="zstd", row_group_size=128_000)

pq.read_metadata("f.parquet")   # rows, row groups, schema
pq.read_schema("f.parquet")

🗂️ Datasets (partitioned)

import pyarrow.dataset as ds

dset = ds.dataset("lake/", format="parquet",
                  partitioning="hive")   # dt=.../region=...
dset.to_table(
    columns=["id", "amt"],
    filter=(ds.field("dt") >= "2026-01-01"))

# write a partitioned dataset
ds.write_dataset(tbl, "out/", format="parquet",
    partitioning=["dt"], existing_data_behavior="overwrite_or_ignore")

Datasets prune partitions & push filters → scan only the files you need across a whole folder tree.

🧮 Compute

import pyarrow.compute as pc

pc.sum(tbl["amt"])      pc.mean(tbl["amt"])
pc.greater(tbl["amt"], 100)      # boolean mask
pc.utf8_upper(tbl["name"])
tbl.group_by("dept").aggregate([("amt", "sum")])
tbl.sort_by([("amt", "descending")])
Interop & storage

🔄 Zero-copy interop

tbl.to_pandas()             # Arrow -> pandas
pa.Table.from_pandas(pdf)   # pandas -> Arrow
pl.from_arrow(tbl)          # -> Polars
duckdb.sql("SELECT * FROM tbl")   # DuckDB reads Arrow directly
# pandas backed by Arrow (less memory, real nulls)
pdf = tbl.to_pandas(types_mapper=pd.ArrowDtype)

These conversions are (mostly) zero-copy — the same buffers, no re-serialization.

☁️ Filesystems

from pyarrow import fs

s3 = fs.S3FileSystem(region="us-east-1")
pq.read_table("bucket/f.parquet", filesystem=s3)
ds.dataset("bucket/lake/", filesystem=s3, format="parquet")
gcs = fs.GcsFileSystem()
local = fs.LocalFileSystem()

⚡ Tips

  • Always pass columns + filters to read only what you need.
  • Tune row_group_size & use zstd/snappy compression.
  • Use datasets for partitioned lakes; use dictionary encoding for repeated strings.
  • Arrow is the glue — prefer it to move data between pandas, Polars, DuckDB, Spark.
