Language (HCL)

🔌 Provider & resource

terraform {
  required_providers {
    aws = { source = "hashicorp/aws", version = "~> 5.0" }
  }
}
provider "aws" { region = var.region }

resource "aws_s3_bucket" "lake" {
  bucket = "my-data-lake"
  tags   = { env = var.env }
}

🔢 Variables & outputs

variable "region" {
  type    = string
  default = "us-east-1"
}
variable "env" { type = string }

output "bucket_name" {
  value = aws_s3_bucket.lake.bucket
}
locals { prefix = "${var.env}-data" }

📥 tfvars & references

# terraform.tfvars
region = "us-east-1"
env    = "prod"

# reference styles
var.env                    # variable
aws_s3_bucket.lake.arn     # resource attr
local.prefix               # local
module.net.vpc_id          # module output
data.aws_caller_identity.me.account_id

🔍 Data sources

data "aws_caller_identity" "me" {}
data "aws_ami" "ubuntu" {
  most_recent = true
  owners      = ["099720109477"]
  filter { name = "name", values = ["ubuntu/*"] }
}
# read existing infra, don't create it
Workflow

⌨️ Core commands

terraform init        # download providers + backend
terraform fmt         # format code
terraform validate    # syntax check
terraform plan        # preview changes
terraform plan -out=tf.plan
terraform apply       # apply (asks to confirm)
terraform apply tf.plan   # apply saved plan
terraform destroy     # tear down

🎯 Targeting & state

terraform apply -target=aws_s3_bucket.lake
terraform apply -var="env=dev" -var-file=dev.tfvars
terraform state list
terraform state show aws_s3_bucket.lake
terraform import aws_s3_bucket.lake my-bucket   # adopt existing
terraform output bucket_name

🗄️ Remote state (backend)

terraform {
  backend "s3" {
    bucket         = "tf-state"
    key            = "data/prod.tfstate"
    region         = "us-east-1"
    dynamodb_table = "tf-locks"   # state locking
    encrypt        = true
  }
}

Shared remote state + locking lets a team apply safely. Never commit *.tfstate (holds secrets).

Reuse & control

📦 Modules

module "network" {
  source = "./modules/network"     # or a registry/git URL
  cidr   = "10.0.0.0/16"
  env    = var.env
}
# use its outputs
resource "aws_instance" "x" {
  subnet_id = module.network.subnet_id
}

🔁 count & for_each

# count -> list, index by [count.index]
resource "aws_instance" "web" {
  count = 3
  tags  = { Name = "web-${count.index}" }
}
# for_each -> map/set, key by [each.key]
resource "aws_s3_bucket" "b" {
  for_each = toset(["raw", "clean"])
  bucket   = "lake-${each.key}"
}
lifecycle { prevent_destroy = true }
depends_on = [aws_iam_role.r]

💡 Best practice

  • Always plan before apply; review the diff. Terraform is declarative — it converges to the config.
  • Use remote state + locking; one state file per environment.
  • Parameterize per env with *.tfvars / workspaces; keep modules small & reusable.
  • fmt + validate in CI; never hand-edit state — use import/state commands.
  • Keep secrets out of code & state comments — source them from a vault/SSM.
