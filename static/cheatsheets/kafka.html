Concepts

🧠 Mental model

  • Topic = named log, split into partitions (the unit of parallelism & ordering).
  • Offset = a message's position in a partition; consumers commit offsets.
  • Order is guaranteed within a partition only — key routes related messages to the same partition.
  • Broker = a server; replication factor copies partitions for durability.
  • Consumer group: each partition is read by exactly one consumer in the group (scale = up to # partitions).

🗂️ Topics CLI

kafka-topics.sh --bootstrap-server localhost:9092 \
  --create --topic orders \
  --partitions 6 --replication-factor 3

kafka-topics.sh --bootstrap-server ... --list
kafka-topics.sh --bootstrap-server ... \
  --describe --topic orders
kafka-topics.sh --bootstrap-server ... \
  --alter --topic orders --partitions 12   # only increase

⌨️ Produce & consume (CLI)

# produce (type messages, Ctrl-D to end)
kafka-console-producer.sh --bootstrap-server ... \
  --topic orders --property parse.key=true \
  --property key.separator=:

# consume from the beginning
kafka-console-consumer.sh --bootstrap-server ... \
  --topic orders --from-beginning \
  --property print.key=true

👥 Consumer groups

kafka-consumer-groups.sh --bootstrap-server ... --list
kafka-consumer-groups.sh --bootstrap-server ... \
  --describe --group etl        # shows LAG per partition
# reset offsets (reprocess)
kafka-consumer-groups.sh --bootstrap-server ... \
  --group etl --topic orders \
  --reset-offsets --to-earliest --execute

Consumer lag = how far behind a group is — the key health metric to alarm on.

Client code

📤 Producer (Python)

from confluent_kafka import Producer
p = Producer({"bootstrap.servers": "localhost:9092",
              "acks": "all",        # wait for replicas
              "enable.idempotence": True})

def cb(err, msg):
    if err: print("fail", err)

p.produce("orders", key="c1", value="{...}", callback=cb)
p.flush()

acks=all + enable.idempotence = no loss, no duplicates from the producer.

📥 Consumer (Python)

from confluent_kafka import Consumer
c = Consumer({"bootstrap.servers": "localhost:9092",
              "group.id": "etl",
              "auto.offset.reset": "earliest",
              "enable.auto.commit": False})
c.subscribe(["orders"])
while True:
    msg = c.poll(1.0)
    if msg and not msg.error():
        handle(msg.value())
        c.commit(msg)      # commit AFTER processing

Commit after processing → at-least-once. Make handling idempotent to dedupe.

🎯 Delivery guarantees

  • At-most-once — commit before processing (may lose on crash).
  • At-least-once — commit after processing (may duplicate). Common default.
  • Exactly-once — idempotent producer + transactions (transactional.id), or an idempotent sink keyed by id.
producer.init_transactions()
producer.begin_transaction(); ...; producer.commit_transaction()

📐 Schema Registry

  • Stores Avro/Protobuf/JSON schemas; messages carry a schema id.
  • Enforces compatibility (backward/forward) so producers & consumers evolve safely.
  • Register a schema before producing; consumers fetch it to deserialize.

⚡ Design & tuning

  • Choose the key so related events share a partition (ordering) but spread evenly (no hot partition).
  • Partitions = max consumer parallelism — size for peak throughput.
  • Retention by time/size; compacted topics keep the latest value per key (changelogs).
  • Processors: Flink / Spark Structured Streaming / Kafka Streams for stateful transforms.
