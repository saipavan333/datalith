⚙️ Dask Cheat Sheet

Parallel & out-of-core Python — scale pandas/NumPy across cores or a cluster with a familiar API, plus custom task graphs with delayed.

Core idea

🧠 Mental model

  • A Dask DataFrame = many pandas DataFrames (partitions) with the same API.
  • Operations are lazy — they build a task graph; .compute() runs it in parallel.
  • Use it when data is bigger than RAM or work is embarrassingly parallel.

📥 Read (partitioned)

import dask.dataframe as dd

df = dd.read_parquet("data/*.parquet")
df = dd.read_csv("data/*.csv", blocksize="128MB")
df.npartitions          # number of chunks
df = df.repartition(partition_size="128MB")
df.to_parquet("out/", write_index=False)

Aim for ~100–256 MB per partition. Many tiny partitions = scheduler overhead; few huge ones = no parallelism / OOM.

🐼 Transform (pandas API)

df[df.amt > 100]
df.assign(net=df.amt - df.fee)
df.groupby("dept").amt.sum()
dd.merge(a, b, on="id", how="left")
df.amt.mean()
df["dept"].value_counts()
# most pandas methods work unchanged

▶️ Lazy → compute

result = df.groupby("dept").amt.sum()   # lazy graph
result.compute()          # run -> pandas result
result.visualize()        # draw the task graph

df = df.persist()         # compute + keep in cluster memory
df.head()                 # small peek (reads 1 partition)

compute() returns a concrete pandas/NumPy object — only call it on results that fit in memory (aggregates), not the whole big frame.

Beyond DataFrames

🧩 dask.delayed

from dask import delayed

@delayed
def load(f): return read(f)
@delayed
def process(x): return transform(x)

tasks = [process(load(f)) for f in files]
results = delayed(sum)(tasks).compute()   # parallel graph

Wrap arbitrary Python functions to parallelize custom pipelines that don't fit the DataFrame API.

🔢 dask.array & bag

import dask.array as da
x = da.random.random((100_000, 1000), chunks=(10_000, 1000))
x.mean(axis=0).compute()      # chunked NumPy

import dask.bag as db
b = db.read_text("logs/*.json").map(json.loads)
b.filter(lambda r: r["ok"]).count().compute()

🖥️ Cluster / scheduler

from dask.distributed import Client, LocalCluster

client = Client()                 # local cluster, all cores
client = Client(LocalCluster(n_workers=4,
                threads_per_worker=2))
# dashboard at http://localhost:8787
print(client.dashboard_link)
client.close()
# scale out: dask-kubernetes, YARN, coiled, etc.

⚠️ Gotchas & when to use

  • Don't .compute() a huge DataFrame — it pulls everything to one machine.
  • set_index / big shuffles are expensive (repartition) — minimize them.
  • Watch partition count & sizes on the dashboard; repartition when skewed.
  • If it fits in RAM, plain pandas/Polars/DuckDB is simpler & faster — reach for Dask when it doesn't, or you need a cluster.
