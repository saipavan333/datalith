DAGs & tasks

📅 Define a DAG

from airflow import DAG
from airflow.operators.bash import BashOperator
from datetime import datetime

with DAG(
    dag_id="daily_etl",
    start_date=datetime(2026, 1, 1),
    schedule="0 6 * * *",     # cron: 06:00 daily
    catchup=False,
    default_args={"retries": 2},
    tags=["etl"],
) as dag:
    t1 = BashOperator(task_id="extract",
                      bash_command="python extract.py")

➡️ Dependencies

t1 >> t2 >> t3            # chain
t1 >> [t2, t3]           # fan-out
[t2, t3] >> t4           # fan-in
t2.set_upstream(t1)      # same as t1 >> t2

from airflow.models.baseoperator import chain
chain(t1, [t2, t3], t4)

🐍 TaskFlow API (modern)

from airflow.decorators import dag, task

@dag(schedule="@daily", start_date=datetime(2026,1,1),
     catchup=False)
def pipeline():
    @task
    def extract(): return [1, 2, 3]
    @task
    def load(rows): print(sum(rows))
    load(extract())      # deps inferred, XCom auto

pipeline()
Scheduling & data passing

⏰ Scheduling

schedule="0 6 * * *"   # cron
schedule="@daily"       # @hourly @weekly @monthly
schedule=timedelta(hours=6)
schedule=None            # manual / triggered only

catchup=True   # backfill from start_date
max_active_runs=1   depends_on_past=True

A run for date D starts after the interval ends. Use {{ ds }} (the logical date), not today().

📦 XCom (pass data)

# push
def extract(**ctx):
    ctx["ti"].xcom_push(key="n", value=42)
# pull
def load(**ctx):
    n = ctx["ti"].xcom_pull(key="n", task_ids="extract")

XCom is for small values (ids, counts, paths) — pass big data via storage (S3/GCS), not XCom.

🕵️ Sensors

from airflow.providers.amazon.aws.sensors.s3 import S3KeySensor

wait = S3KeySensor(
    task_id="wait_file",
    bucket_key="s3://bucket/dt={{ ds }}/_SUCCESS",
    mode="reschedule",   # free the worker slot while waiting
    poke_interval=60, timeout=3600)

Use mode="reschedule" for long waits so a sensor doesn't hog a worker.

💡 Best practice

  • Tasks must be idempotent & keyed by {{ ds }} → safe retries/backfills.
  • Keep DAG files light — no heavy work at parse time (top level).
  • One unit of work per task; make failures small & re-runnable.
  • Airflow orchestrates (triggers & monitors) — do the compute in Spark/warehouse/dbt.
