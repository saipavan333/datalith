Building blocks

⚓ Anchors & boundaries

^        # start of string/line
$        # end of string/line
\b       # word boundary
\B       # non-boundary
\A  \Z   # start / end of whole string

^\d+$    # entire string is digits

🔡 Character classes

.        # any char (except newline)
\d \D    # digit / non-digit
\w \W    # word [A-Za-z0-9_] / non-word
\s \S    # whitespace / non-whitespace
[abc]    # a, b, or c
[^abc]   # NOT a, b, c
[a-z0-9] # ranges
\.  \*   # escape a literal metachar

🔢 Quantifiers

*        # 0 or more
+        # 1 or more
?        # 0 or 1 (optional)
{3}      # exactly 3
{2,5}    # 2 to 5
{2,}     # 2 or more

*?  +?  ??   # lazy (fewest) vs greedy default

Greedy .* grabs as much as possible; add ? for lazy .*?.

( ) Groups & alternation

(abc)      # capture group 1
(?:abc)    # group, no capture
(?P<year>\d{4})  # named group (Python)
a|b        # a OR b
(cat|dog)s # cats or dogs
\1         # backreference to group 1

👀 Lookaround

(?=foo)    # lookahead: followed by foo
(?!foo)    # negative lookahead
(?<=foo)   # lookbehind: preceded by foo
(?<!foo)   # negative lookbehind

\d+(?= USD)   # number before " USD"
(?<=\$)\d+    # number after "$"

🚩 Flags

i   # ignore case
m   # multiline (^ $ per line)
s   # dotall (. matches newline)
x   # verbose (ignore whitespace)
g   # global (all matches - tool-level)

# inline: (?i) at start, or re.IGNORECASE
Patterns & usage

📋 Common patterns

# email (pragmatic)
[\w.+-]+@[\w-]+\.[\w.-]+

# ISO date YYYY-MM-DD
\d{4}-\d{2}-\d{2}

# IPv4 (loose)
(\d{1,3}\.){3}\d{1,3}

# price / integer with commas
\d{1,3}(,\d{3})*(\.\d+)?

# whitespace collapse
\s+

🐍 Python re

import re

re.search(p, s)        # first match anywhere
re.match(p, s)         # match at start
re.findall(p, s)       # list of matches
re.finditer(p, s)      # iterator of match objs
re.sub(p, repl, s)     # replace all
re.split(p, s)

m = re.search(r"(?P<y>\d{4})", s)
m.group("y")   m.group(1)   m.start()
pat = re.compile(p, re.I)   # reuse compiled

Use raw strings r"\d+" so \ isn't eaten by Python.

🐚 grep & sed

# extended regex
grep -E "[0-9]{3}-[0-9]{4}" f
grep -oE "\w+@\w+" f     # only matches
grep -P "(?<=id=)\d+" f  # Perl (lookbehind)

# sed capture + backref
sed -E "s/(\w+)@(\w+)/\2-\1/" f

Basic grep needs \( \); -E uses plain ( ) { }.

🗄️ Regex in SQL

# Postgres / Snowflake / BigQuery
s ~ '^A.*z$'                 # PG match
REGEXP_LIKE(s, '\\d+')         # Snowflake
REGEXP_REPLACE(s, '\\s+', ' ')
REGEXP_EXTRACT(s, r'(\d{4})')   # BigQuery
REGEXP_SUBSTR(s, '[0-9]+')

⚠️ Gotchas

  • Escape literals: . * + ? ( ) [ ] { } ^ $ | \.
  • . excludes newline unless the s (dotall) flag is set.
  • Greedy by default — <.*> over-matches; use <.*?>.
  • Catastrophic backtracking: avoid nested quantifiers like (a+)+.
  • Validate hard formats (emails, phones) with a library when you can.
