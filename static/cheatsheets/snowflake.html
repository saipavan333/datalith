Architecture & context

🏔️ Architecture

  • 3 layers: cloud services (optimizer, metadata, security) · compute (virtual warehouses) · storage (columnar micro-partitions) — scale independently.
  • Micro-partitions (~50–500 MB, immutable) carry min/max metadata → automatic pruning.
  • Caching: result cache (24 h, free) → warehouse SSD → remote storage.

⚙️ Warehouses & context

CREATE WAREHOUSE wh
  WAREHOUSE_SIZE = 'MEDIUM'
  AUTO_SUSPEND = 60      -- seconds idle
  AUTO_RESUME  = TRUE
  MIN_CLUSTER_COUNT=1 MAX_CLUSTER_COUNT=3; -- multi-cluster

USE WAREHOUSE wh;
USE DATABASE db; USE SCHEMA sch; USE ROLE eng;
ALTER WAREHOUSE wh SUSPEND;

Size up (bigger warehouse) for slow queries; add clusters for concurrency. Billed per-second while running.

Loading data

📥 Stages, formats & COPY

CREATE FILE FORMAT ff TYPE=CSV
  FIELD_OPTIONALLY_ENCLOSED_BY='"' SKIP_HEADER=1;

CREATE STAGE my_stage
  URL='s3://bucket/path'
  STORAGE_INTEGRATION=s3_int
  FILE_FORMAT=ff;

COPY INTO t FROM @my_stage
  FILE_FORMAT=(FORMAT_NAME=ff)
  ON_ERROR='CONTINUE';
LIST @my_stage;   PUT file://local @my_stage;

🚰 Snowpipe (auto-ingest)

CREATE PIPE sales_pipe AUTO_INGEST=TRUE AS
  COPY INTO sales
  FROM @my_stage
  FILE_FORMAT=(TYPE=PARQUET);

-- point cloud event notifications at the pipe's ARN
SELECT SYSTEM$PIPE_STATUS('sales_pipe');

Serverless, continuous, event-driven ingest. Dedups files for ~14 days; no ordering guarantee.

🔄 Streams & Tasks (pipelines)

-- Stream = CDC on a table (inserts/updates/deletes)
CREATE STREAM s ON TABLE src;

CREATE TASK load_task
  WAREHOUSE=wh
  SCHEDULE='1 MINUTE'
  WHEN SYSTEM$STREAM_HAS_DATA('s') AS
  MERGE INTO tgt USING s ON ...;

ALTER TASK load_task RESUME;   -- tasks start suspended

🧊 Dynamic Tables

CREATE DYNAMIC TABLE dt
  TARGET_LAG = '5 minutes'
  WAREHOUSE = wh AS
  SELECT dept, SUM(amt) t FROM orders GROUP BY dept;

Declarative, auto-incremental transforms — you define the result & freshness (lag); Snowflake maintains it. Simpler than Streams+Tasks.

Time travel, security, semi-structured

⏱️ Time Travel & Clone

SELECT * FROM t AT(OFFSET => -3600);   -- 1h ago
SELECT * FROM t BEFORE(STATEMENT => '<query_id>');
UNDROP TABLE t;

-- zero-copy clone (instant, metadata only)
CREATE TABLE t_dev CLONE t;
CREATE DATABASE dev CLONE prod;

Time Travel up to 90 days (Enterprise); Fail-safe adds 7 more (recovery only).

🔐 RBAC & grants

CREATE ROLE analyst;
GRANT USAGE ON WAREHOUSE wh TO ROLE analyst;
GRANT USAGE ON DATABASE db TO ROLE analyst;
GRANT SELECT ON ALL TABLES IN SCHEMA db.sch TO ROLE analyst;
GRANT SELECT ON FUTURE TABLES IN SCHEMA db.sch TO ROLE analyst;
GRANT ROLE analyst TO USER ada;

Grant to roles, not users. Future grants cover tables created later. Role hierarchy rolls up privileges.

⚡ Performance & tuning

  • Add a CLUSTER BY (col) only on very large, filtered tables; check SYSTEM$CLUSTERING_INFORMATION.
  • Prune via WHERE on high-cardinality/date columns (micro-partition min/max).
  • Right-size the warehouse; use a separate WH per workload; enable auto-suspend.
  • Read Query Profile for spilling / exploding joins; use RESULT_SCAN to reuse results.
  • SELECT ... FROM t SAMPLE (1) to sample; avoid SELECT *.
