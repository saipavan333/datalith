🟦 GCP CLI Cheat Sheet

Google Cloud from the terminal — gcloud, gsutil (GCS), bq (BigQuery), Pub/Sub, Dataflow, Dataproc, Composer and IAM.

Setup

🔑 gcloud config & auth

gcloud init
gcloud auth login
gcloud auth application-default login   # for SDKs
gcloud config set project my-proj
gcloud config set compute/region us-central1
gcloud config list
gcloud projects list

🎛️ Output & filter

--format=json | table | value(...)
--filter="status=RUNNING"
gcloud compute instances list \
  --format="value(name,zone)"
gcloud <svc> --help
Storage & BigQuery

🪣 gsutil (GCS)

gsutil ls gs://bucket/prefix/
gsutil cp file.csv gs://bucket/path/
gsutil cp gs://bucket/f.csv .
gsutil -m cp -r ./dir gs://bucket/dir/   # -m = parallel
gsutil rsync -r ./local gs://bucket/dir/
gsutil rm -r gs://bucket/old/
gsutil du -sh gs://bucket/
# newer: gcloud storage cp ... (faster)

🔎 bq (BigQuery)

bq query --use_legacy_sql=false 'SELECT 1'
bq ls mydata            bq show mydata.t
bq mk --dataset US:mydata
bq load --source_format=PARQUET \
  mydata.t gs://bucket/*.parquet
bq extract mydata.t gs://bucket/out-*.csv
bq query --dry_run --use_legacy_sql=false '…'   # cost check

📨 Pub/Sub

gcloud pubsub topics create orders
gcloud pubsub subscriptions create sub1 --topic orders
gcloud pubsub topics publish orders --message '{"id":1}'
gcloud pubsub subscriptions pull sub1 --auto-ack --limit 10

A subscription only receives messages published after it exists (or within retention via seek).

Processing & governance

🌊 Dataflow (Beam)

gcloud dataflow jobs list --status=active
gcloud dataflow jobs run my-job \
  --gcs-location gs://dataflow-templates/.../Word_Count \
  --region us-central1 \
  --parameters inputFile=gs://in,output=gs://out
gcloud dataflow jobs cancel <JOB_ID>

⚡ Dataproc (Spark)

gcloud dataproc clusters create c1 --region us-central1 \
  --num-workers 3 --image-version 2.2
gcloud dataproc jobs submit pyspark job.py \
  --cluster c1 --region us-central1
gcloud dataproc clusters delete c1 --region us-central1
# Composer = managed Airflow
gcloud composer environments list --locations us-central1

🔐 IAM

gcloud iam service-accounts create etl-sa
gcloud projects add-iam-policy-binding my-proj \
  --member="serviceAccount:etl-sa@my-proj.iam.gserviceaccount.com" \
  --role="roles/bigquery.dataEditor"
gcloud iam service-accounts keys create key.json \
  --iam-account=etl-sa@my-proj.iam.gserviceaccount.com

Prefer workload identity / ADC over downloading key files.

💡 Tips

  • gcloud storage is the newer, faster replacement for many gsutil commands.
  • Set project/region once in config to shorten every command.
  • Use --dry_run (bq) & least-privilege service accounts for pipelines.
  • Dataplex / Data Catalog handle governance & metadata across GCS + BigQuery.
