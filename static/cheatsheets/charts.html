🧭 What question am I answering?

Comparison ....... bar / grouped bar

Distribution ..... histogram / box / violin

Chart type follows the question & data, not decoration. Name the takeaway first, then pick the chart that shows it fastest.

By purpose

📊 Comparison

  • Bar / column — compare values across categories. The default; sort by value.
  • Grouped bar — compare a few series per category (≤ ~3 series).
  • Stacked bar — compare totals and composition (but hard to compare inner segments).
  • Bullet — a value vs a target/threshold (KPI).

Bars must start at zero — a cut axis exaggerates differences.

📈 Trend over time

  • Line — continuous change over time; best for many points / multiple series.
  • Area — a line with emphasis on magnitude/volume.
  • Stacked area — how composition evolves over time.
  • Candlestick / OHLC — financial open-high-low-close.

Time goes on the x-axis, left→right. Use a line (not bars) when order/continuity matters.

🔔 Distribution

Histogram    ▁▃▅▇▅▃▁   shape of one variable
Box plot     |--[ | ]--o   median, IQR, outliers
Violin       density + box combined
  • Histogram — the shape/spread of one numeric variable (tune bin count).
  • Box — median, quartiles, outliers; great to compare groups compactly.
  • Violin / KDE — full density shape (bimodality) — richer than a box.
  • ECDF — cumulative — good for percentiles & comparisons.

🔗 Relationship

  • Scatter — correlation between two numeric variables.
  • Bubble — scatter + a third variable as size.
  • Heatmap — a matrix of values (e.g. correlation, cohort, calendar).
  • Hexbin / 2-D density — scatter when points overlap (big data).

Correlation ≠ causation — a scatter shows association, not cause.

🥧 Part-to-whole

  • Stacked bar (100%) — composition across categories. Usually the best choice.
  • Treemap — hierarchical parts, many categories, size = value.
  • Pie / donut — only for 2–3 slices; humans compare angles poorly.

Prefer a bar over a pie for anything beyond a couple of slices.

🏆 Ranking

  • Ordered bar — top-N; sort descending, label clearly.
  • Lollipop — a cleaner ordered bar for many categories.
  • Slope / bump — rank change between two periods.

🌊 Flow & process

  • Sankey — flows/volumes between stages (funnels, ETL, budgets).
  • Funnel — drop-off through sequential steps (signup → purchase).
  • Waterfall — how a total is built up/down by contributions.

🗺️ Geospatial & KPI

  • Choropleth — a metric shaded by region (normalize by population/area!).
  • Symbol / bubble map — point locations sized by value.
  • Big number — a single headline KPI (+ sparkline / delta).
  • Gauge — value vs range (use sparingly; a bullet is clearer).
Principles

✅ Make it honest & clear

  • Zero baseline for bars; be careful with truncated axes.
  • One clear message per chart; put it in the title.
  • Sort by value (not alphabetically) unless order has meaning.
  • Direct-label lines; minimize legends & gridlines (cut chartjunk).
  • Avoid dual y-axes (misleading), 3-D, and rainbow palettes.
  • Color with intent: sequential for magnitude, diverging for +/−, categorical for groups; check colorblind-safety.
  • Aggregate huge data (plotting millions of points is slow & unreadable).

🎯 Quick decision guide

"compare categories"      -> bar
"over time"               -> line
"spread / outliers"       -> box / histogram
"x vs y related?"         -> scatter
"share of total"          -> stacked bar / treemap
"drop-off by step"        -> funnel
"where (map)"             -> choropleth
"one KPI"                 -> big number
