Images & containers

📦 Core commands

docker images                 # list images
docker ps          docker ps -a   # running / all
docker pull python:3.12-slim
docker run --rm hello-world
docker stop <id>    docker rm <id>
docker rmi <image>
docker logs -f <id>
docker exec -it <id> bash     # shell into container

▶️ docker run flags

docker run \
  -d                     # detached
  --name etl             # name it
  -p 8080:80             # host:container port
  -v $(pwd)/data:/data   # bind mount
  -e DB_URL="..."        # env var
  --env-file .env
  --rm                   # remove on exit
  myimage:tag  python job.py

📝 Dockerfile

FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
ENV PYTHONUNBUFFERED=1
EXPOSE 8000
ENTRYPOINT ["python"]
CMD ["job.py"]

Copy requirements first, install, then copy code — so the pip layer caches across code changes.

🏗️ Build & tag

docker build -t myimg:1.0 .
docker build -t myimg:1.0 -f Dockerfile.prod .
docker tag myimg:1.0 registry/acct/myimg:1.0
docker push registry/acct/myimg:1.0
docker login registry.example.com
docker history myimg:1.0     # layer sizes
Compose & ops

🧩 docker compose

# compose.yaml
services:
  db:
    image: postgres:16
    environment: { POSTGRES_PASSWORD: pw }
    ports: ["5432:5432"]
    volumes: ["pgdata:/var/lib/postgresql/data"]
  app:
    build: .
    depends_on: [db]
volumes: { pgdata: {} }
docker compose up -d     docker compose down
docker compose logs -f app     docker compose ps

💾 Volumes & networks

docker volume ls        docker volume create data
docker volume rm data
# bind mount (host path) vs named volume (managed)
docker network ls
docker network create mynet
docker run --network mynet ...   # containers resolve by name

🧹 Cleanup

docker system df           # disk usage
docker container prune     # remove stopped
docker image prune -a      # unused images
docker volume prune
docker system prune -a --volumes   # reclaim everything (!)

💡 Best practice

  • Use small bases (-slim, alpine) & multi-stage builds to shrink images.
  • Add a .dockerignore (skip .git, data, venvs).
  • Pin versions (python:3.12-slim, not latest) for reproducibility.
  • One process per container; pass config via env; don't bake secrets into images.
  • Containers are ephemeral — persist data in volumes, not the container layer.
