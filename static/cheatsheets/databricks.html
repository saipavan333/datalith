Delta Lake

🔺 Create & write Delta

%sql
CREATE TABLE t (id BIGINT, amt DOUBLE) USING DELTA
  PARTITIONED BY (dt);

# append / overwrite
df.write.format("delta").mode("append").saveAsTable("t")
df.write.mode("overwrite").option("replaceWhere",
             "dt = '2026-01-01'").saveAsTable("t")

Delta = Parquet + a transaction log (_delta_log) → ACID, versions, schema enforcement.

🔀 MERGE (upsert)

%sql
MERGE INTO tgt t
USING updates u ON t.id = u.id
WHEN MATCHED THEN UPDATE SET *
WHEN NOT MATCHED THEN INSERT *
WHEN NOT MATCHED BY SOURCE THEN DELETE;   -- full sync

🧹 OPTIMIZE, time travel, CDF

%sql
OPTIMIZE t ZORDER BY (user_id);   -- compact + co-locate
VACUUM t RETAIN 168 HOURS;      -- purge old files

SELECT * FROM t VERSION AS OF 12;
SELECT * FROM t TIMESTAMP AS OF '2026-01-01';
DESCRIBE HISTORY t;

# Change Data Feed
ALTER TABLE t SET TBLPROPERTIES(delta.enableChangeDataFeed=true);
SELECT * FROM table_changes('t', 10);
Ingest, stream & govern

📡 Auto Loader

(spark.readStream.format("cloudFiles")
   .option("cloudFiles.format", "json")
   .option("cloudFiles.schemaLocation", "/chk/schema")
   .load("/mnt/raw/")
 .writeStream
   .option("checkpointLocation", "/chk/bronze")
   .trigger(availableNow=True)
   .toTable("bronze"))

Incrementally & idempotently ingests new files from cloud storage; tracks state in the checkpoint.

🌊 Structured Streaming

(spark.readStream.table("bronze")
   .withWatermark("ts", "10 minutes")
   .groupBy(F.window("ts", "5 minutes"), "dept")
   .agg(F.sum("amt"))
 .writeStream.format("delta")
   .option("checkpointLocation", "/chk/agg")
   .outputMode("append").toTable("gold"))

Checkpoint + idempotent Delta sink = exactly-once.

🏗️ Delta Live Tables (DLT)

import dlt

@dlt.table
def bronze():
    return spark.readStream.format("cloudFiles")...load("/raw")

@dlt.table
@dlt.expect_or_drop("valid_amt", "amt >= 0")
def silver():
    return dlt.read_stream("bronze").where("id IS NOT NULL")

Declarative pipelines: dependencies, quality expectations, and orchestration are managed for you.

🧰 dbutils, widgets & jobs

dbutils.fs.ls("/mnt")   dbutils.fs.cp(a, b)

dbutils.widgets.text("dt", "2026-01-01")

Jobs orchestrate multi-task workflows on a schedule. Use job clusters (ephemeral) for scheduled runs, not all-purpose clusters.

⚡ Tuning

  • Photon (vectorized engine) for SQL/ETL speed; enable on the cluster.
  • OPTIMIZE + ZORDER on filter columns; keep files ~128–256 MB.
  • Enable AQE (default) for skew joins & dynamic partitions; broadcast small sides.
  • Partition by low-cardinality columns (e.g. date) — not high-cardinality ids.
  • Medallion: bronze (raw) → silver (clean) → gold (aggregated/serving).
