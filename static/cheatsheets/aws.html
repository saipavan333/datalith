Setup & basics

🔑 Configure & auth

aws configure                 # key, secret, region, format
aws configure --profile prod
aws sts get-caller-identity   # who am I
export AWS_PROFILE=prod
export AWS_REGION=us-east-1
aws sso login --profile prod

🎛️ Output & query

--output json | table | text
--region us-east-1     --profile prod
# JMESPath filtering
aws s3api list-buckets \
  --query "Buckets[].Name" --output text
aws ec2 describe-instances \
  --query "Reservations[].Instances[].InstanceId"
Storage & catalog

🪣 S3

aws s3 ls s3://bucket/prefix/
aws s3 cp file.csv s3://bucket/path/
aws s3 cp s3://bucket/f.csv .           # download
aws s3 sync ./local s3://bucket/dir/    # incremental
aws s3 sync s3://bucket/dir/ ./local --delete
aws s3 rm s3://bucket/old/ --recursive
aws s3 presign s3://bucket/f --expires-in 3600
# low-level api
aws s3api list-objects-v2 --bucket b --prefix p/ \
  --query "Contents[].Key"

🧰 Glue

aws glue start-job-run --job-name etl \
  --arguments '{"--dt":"2026-01-01"}'
aws glue get-job-run --job-name etl --run-id jr_..
aws glue start-crawler --name my_crawler
aws glue get-tables --database-name lake \
  --query "TableList[].Name"
aws glue get-databases

Glue Data Catalog = the Hive metastore Athena/Redshift Spectrum/EMR all share.

🔎 Athena

qid=$(aws athena start-query-execution \
  --query-string "SELECT count(*) FROM db.t" \
  --result-configuration OutputLocation=s3://bucket/aq/ \
  --query-execution-context Database=db \
  --query "QueryExecutionId" --output text)

aws athena get-query-execution --query-execution-id $qid \
  --query "QueryExecution.Status.State"
aws athena get-query-results --query-execution-id $qid
Warehouse, streams & orchestration

🟥 Redshift Data API

aws redshift-data execute-statement \
  --cluster-identifier dw --database prod \
  --db-user etl --sql "SELECT count(*) FROM sales"
aws redshift-data describe-statement --id <id>
aws redshift-data get-statement-result --id <id>

Serverless SQL calls (no JDBC/driver) — handy from Lambda/Step Functions.

🌊 Kinesis

aws kinesis put-record --stream-name orders \
  --partition-key c1 --data "$(echo '{...}' | base64)"
aws kinesis describe-stream-summary --stream-name orders
aws firehose put-record --delivery-stream-name s3-ds \
  --record Data="..."

🔗 Step Functions & EMR

aws stepfunctions start-execution \
  --state-machine-arn arn:aws:states:... \
  --input '{"dt":"2026-01-01"}'
aws stepfunctions describe-execution --execution-arn ...

aws emr create-cluster --release-label emr-7.0.0 \
  --applications Name=Spark --instance-type m5.xlarge \
  --instance-count 3 --use-default-roles

🔐 IAM & Lambda

aws iam list-roles --query "Roles[].RoleName"
aws iam attach-role-policy --role-name r \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

aws lambda invoke --function-name f \
  --payload '{"k":1}' out.json
aws logs tail /aws/lambda/f --follow

💡 Tips

  • Use named profiles (--profile) & SSO instead of long-lived keys.
  • --dry-run where supported; --no-paginate/--max-items for big lists.
  • Prefer roles (instance/task roles) over embedding credentials in pipelines.
  • aws help / aws s3 help — every service is self-documenting.
