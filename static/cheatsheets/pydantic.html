Models

🧱 Basic model

from pydantic import BaseModel
from datetime import datetime

class User(BaseModel):
    id: int
    name: str
    email: str
    active: bool = True            # default
    created: datetime | None = None # optional

u = User(id="1", name="Ada", email="a@x.com")
u.id        # 1  (coerced str -> int)
u.model_dump()   # -> dict

Pydantic coerces & validates on construction — bad types raise ValidationError.

🏷️ Field types

from typing import Optional, Literal
from pydantic import EmailStr, HttpUrl, Field

class Order(BaseModel):
    id: int
    tags: list[str] = []
    meta: dict[str, str] = {}
    status: Literal["new", "paid"] = "new"
    email: EmailStr
    url: HttpUrl | None = None
    amount: float

📐 Field constraints

from pydantic import Field

class Product(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    price: float = Field(gt=0)              # > 0
    qty: int = Field(ge=0, le=1000)         # 0..1000
    sku: str = Field(pattern=r"^[A-Z]{3}-\d+$")
    note: str = Field(default="", alias="comment")

gt/ge/lt/le numbers · min_length/max_length strings/lists · pattern regex · alias for source key names.

Validation

✅ Validators

from pydantic import field_validator, model_validator

class Signup(BaseModel):
    email: str
    pw: str
    pw2: str

    @field_validator("email")
    @classmethod
    def lower(cls, v): return v.strip().lower()

    @model_validator(mode="after")
    def match(self):
        if self.pw != self.pw2:
            raise ValueError("passwords differ")
        return self

🪆 Nested models

class Address(BaseModel):
    city: str
    zip: str

class Customer(BaseModel):
    name: str
    address: Address              # nested
    orders: list[Order] = []

c = Customer.model_validate({
    "name": "Ada",
    "address": {"city": "NYC", "zip": "10001"}})
c.address.city      # "NYC"

🔄 Parse & serialize

User.model_validate(some_dict)      # dict -> model

u.model_dump()                     # -> dict
u.model_dump(exclude_none=True, by_alias=True)
u.model_dump_json(indent=2)        # -> json str

🛠️ Error handling

from pydantic import ValidationError

try:
    User(id="x", name=1, email="")
except ValidationError as e:
    print(e.errors())    # list of {loc, msg, type}
    print(e.json())

Catch ValidationError at ingestion; route bad records to a dead-letter/quarantine table.

⚙️ Config & settings

from pydantic import ConfigDict

class Strict(BaseModel):
    model_config = ConfigDict(
        extra="forbid",      # reject unknown fields
        frozen=True,          # immutable
        str_strip_whitespace=True)

# app config from env (pip install pydantic-settings)
from pydantic_settings import BaseSettings
class Settings(BaseSettings):
    db_url: str
    workers: int = 4
cfg = Settings()   # reads env vars / .env

💡 Where it fits

  • Validate API responses, file rows & configs at the boundary before they enter pipelines.
  • Coercion turns messy strings into typed values; constraints catch bad data early.
  • Pairs with FastAPI (request/response models) & pydantic-settings (config).
  • For big tabular data, validate with frameworks like Pandera/Great Expectations instead.
