Σ GroupBy & aggregate

(df.groupBy("dept")
   .agg(F.sum("amt").alias("total"),

🪟 Window functions

w = W.partitionBy("dept").orderBy(F.desc("amt"))

df.withColumn("rn", F.row_number().over(w))
  .withColumn("rk", F.dense_rank().over(w))
  .withColumn("prev", F.lag("amt", 1).over(w))

wr = (W.partitionBy("dept").orderBy("ts")
       .rowsBetween(-6, 0))          # 7-row window
df.withColumn("roll", F.avg("amt").over(wr))
# dedup: keep rn==1
df.withColumn("rn", F.row_number().over(w)).filter("rn = 1")

🧩 UDFs (last resort)

from pyspark.sql.types import StringType

@F.udf(StringType())
def clean(s): return (s or "").strip().lower()
df.withColumn("c", clean("name"))

# vectorized pandas UDF (faster)
@F.pandas_udf("double")
def norm(x): return (x - x.mean()) / x.std()

Prefer built-in F.* functions — UDFs skip Catalyst optimization & are slower.

Performance

🔀 Shuffle, skew & partitions

df.repartition(200, "key")   # wide, even
df.coalesce(1)                # narrow, merge (no shuffle)
df.rdd.getNumPartitions()

# Adaptive Query Execution (on by default 3.x)
spark.conf.set("spark.sql.adaptive.enabled", True)
spark.conf.set("spark.sql.adaptive.skewJoin.enabled", True)
  • Shuffle (join/groupBy/distinct) is the main cost — filter & select before it.
  • Skew: salt the hot key, or rely on AQE skew join.
  • Target ~128–256 MB per partition; avoid tiny files.

💾 Cache & broadcast

df.cache()   df.persist()   df.unpersist()
# reuse a DataFrame across multiple actions

F.broadcast(small_df)   # force broadcast join
spark.conf.set("spark.sql.autoBroadcastJoinThreshold",
               10*1024*1024)   # 10MB auto-broadcast

Lazy: transformations build a plan; actions (count/collect/write/show) run it. Cache only if reused.

🧠 Gotchas

  • collect() pulls all rows to the driver — OOM risk on big data.
  • Watch skew (one slow task) and small files on write.
  • Column pruning + predicate pushdown work best on Parquet/Delta.
  • Prefer DataFrame/SQL over RDDs (Catalyst + Tungsten optimize them).
