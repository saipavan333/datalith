Model & DDL

🟦 How it bills

  • Serverless MPP; on-demand billing = bytes scanned (or slots for capacity).
  • Storage & compute are separate; no indexes — control cost via partition + cluster and narrow SELECT.
  • Fully-qualified name: project.dataset.table.

🧱 Partition & cluster

CREATE TABLE events (ts TIMESTAMP, uid INT64, ...)
PARTITION BY DATE(ts)
CLUSTER BY uid, event_type
OPTIONS(require_partition_filter=true);

-- prune: filter the partition column
SELECT * FROM events
WHERE DATE(ts) BETWEEN '2026-01-01' AND '2026-01-07';

Partition by date to scan fewer bytes; cluster by common filter columns to skip blocks. require_partition_filter blocks full scans.

-- explode an array
SELECT id, tag
FROM t, UNNEST(tags) AS tag;

-- build / aggregate arrays
ARRAY_AGG(x ORDER BY ts LIMIT 5)
ARRAY_LENGTH(tags)   tags[OFFSET(0)]
STRUCT(1 AS a, 'x' AS b)
SELECT cust.name FROM t;   -- dotted struct access

🔀 MERGE & DML

MERGE mydata.tgt t
USING mydata.src s ON t.id = s.id
WHEN MATCHED THEN UPDATE SET amt = s.amt
WHEN NOT MATCHED THEN INSERT (id, amt)
     VALUES (s.id, s.amt);

DELETE FROM t WHERE DATE(ts) < '2025-01-01';
UPDATE t SET amt = 0 WHERE amt IS NULL;

🧠 Handy SQL

SAFE_CAST(s AS INT64)      -- NULL on failure
SAFE_DIVIDE(a, b)
PARSE_DATE('%Y-%m-%d', s)
DATE_TRUNC(dt, MONTH)   DATE_DIFF(a, b, DAY)
APPROX_COUNT_DISTINCT(uid)   -- HyperLogLog
QUALIFY ROW_NUMBER() OVER(PARTITION BY id
         ORDER BY ts DESC) = 1   -- dedup, no subquery
CLI, load & cost

⌨️ bq CLI

bq query --use_legacy_sql=false 'SELECT 1'
bq ls mydata            bq show mydata.orders
bq mk --dataset US:mydata
bq load --source_format=PARQUET \
   mydata.t gs://bucket/*.parquet
bq extract mydata.t gs://bucket/out-*.csv
bq rm -t mydata.t

💸 Cost controls

-- estimate bytes without running
bq query --dry_run --use_legacy_sql=false '…'
-- cap bytes billed
bq query --maximum_bytes_billed=1000000000 '…'
  • Never SELECT * — name columns (columnar scan cost).
  • Filter the partition column; preview with tablesample.
  • Materialize hot results; use BI Engine/cached results.

🔗 External & scheduled

-- query files in GCS directly
CREATE EXTERNAL TABLE ext
OPTIONS(format='PARQUET',
        uris=['gs://bucket/*.parquet']);

-- scheduled query (recurring)
bq mk --transfer_config --data_source=scheduled_query \
  --schedule='every 24 hours' --target_dataset=mydata \
  --params='{"query":"…","destination_table_name_template":"daily"}'

CREATE MODEL mydata.churn
OPTIONS(model_type='LOGISTIC_REG',
        input_label_cols=['churned']) AS
SELECT * FROM mydata.training;

SELECT * FROM ML.PREDICT(MODEL mydata.churn,
       (SELECT * FROM mydata.new));
SELECT * FROM ML.EVALUATE(MODEL mydata.churn);

Train & serve models in SQL — no data movement.
