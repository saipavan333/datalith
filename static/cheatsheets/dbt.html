🔗 Models & ref

-- models/marts/fct_orders.sql
{{ config(materialized='table') }}

select
  o.id, o.amt, c.name
from {{ ref('stg_orders') }} o        -- model dep
join {{ source('shop', 'customers') }} c  -- raw source
  on o.cid = c.id

ref()/source() build the DAG & lineage automatically — never hard-code table names.

➕ Incremental models

{{ config(materialized='incremental',
          unique_key='id',
          incremental_strategy='merge') }}

select * from {{ ref('stg_orders') }}
{% if is_incremental() %}
  where updated_at > (select max(updated_at) from {{ this }})
{% endif %}

Only processes new/changed rows; {{ this }} = the model's own table. Full refresh: dbt run --full-refresh.

Sources, tests & snapshots

📇 Sources & freshness

# models/_schema.yml
sources:
  - name: shop
    schema: raw
    tables:
      - name: customers
        freshness:
          warn_after: {count: 12, period: hour}
          error_after: {count: 24, period: hour}
        loaded_at_field: _loaded_at
dbt source freshness

✅ Tests

# _schema.yml — generic tests
models:
  - name: fct_orders
    columns:
      - name: id
        tests: [unique, not_null]
      - name: cid
        tests:
          - relationships:
              to: ref('dim_customer')
              field: id
      - name: status
        tests:
          - accepted_values: {values: ['new','paid']}

Singular tests = any .sql in tests/ that returns failing rows. Run: dbt test.

📸 Snapshots (SCD2)

{% snapshot cust_snap %}
{{ config(target_schema='snapshots',
   unique_key='id',
   strategy='timestamp',
   updated_at='updated_at') }}
  select * from {{ source('shop','customers') }}
{% endsnapshot %}

Captures history (valid_from/valid_to) — SCD2 without hand-writing it. Run: dbt snapshot.

CLI & docs

⌨️ CLI commands

dbt debug            # check connection
dbt run              # build all models
dbt run -s fct_orders+     # model + downstream
dbt run -s tag:daily       # by selector/tag
dbt build            # run + test + snapshot + seed
dbt test             dbt snapshot     dbt seed
dbt run --full-refresh
dbt docs generate && dbt docs serve

🎯 Node selection

-s model_name        # just this model
-s model+            # + all downstream
-s +model            # + all upstream
-s +model+           # full lineage
-s staging.*         # a folder/package
-s tag:nightly       # by tag
--exclude model_x    # skip some
-s state:modified    # CI: changed only

💡 Best practice

  • Staging (1:1 cleanup) → marts (business models). One folder per layer.
  • Everything via ref()/source() — never hard-code names.
  • Test keys (unique + not_null) & relationships; block merges on failures in CI.
  • Runs in-warehouse = ELT; dbt owns transforms, an orchestrator triggers it.
