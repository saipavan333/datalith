Connect & query

🔌 Connect

import duckdb

duckdb.sql("SELECT 42")              # in-memory, one-off
con = duckdb.connect("my.db")          # persistent file
con = duckdb.connect()                  # in-memory session
con.sql("SELECT * FROM t").show()
con.execute("INSERT INTO t VALUES (?)", [1])

📂 Query files directly

-- no load step; SQL straight over files
duckdb.sql("SELECT * FROM 'data/*.parquet'")
duckdb.sql("SELECT * FROM read_csv_auto('f.csv')")
duckdb.sql("""
  SELECT dept, sum(amt) t
  FROM read_parquet('s3://bucket/*.parquet')
  GROUP BY dept
""")
duckdb.sql("SELECT * FROM read_json_auto('f.json')")

Reads only the columns/row-groups it needs from Parquet — fast & cheap on big files.

🐼 SQL on DataFrames

import pandas as pd, polars as pl
orders = pd.read_parquet("orders.parquet")

# reference the Python variable by name in SQL
duckdb.sql("SELECT dept, count(*) FROM orders GROUP BY dept")

# works with Polars & Arrow too, zero-copy
res = duckdb.sql("SELECT * FROM orders WHERE amt > 100")
res.df()      # -> pandas
res.pl()      # -> polars
res.arrow()   res.fetchall()

🏗️ Tables & DML

con.sql("CREATE TABLE t AS SELECT * FROM 'f.parquet'")
con.sql("CREATE OR REPLACE TABLE t (id INT, amt DOUBLE)")
con.sql("INSERT INTO t SELECT * FROM read_csv_auto('f.csv')")
con.sql("CREATE VIEW v AS SELECT ...")
-- persist a query to a file
con.sql("COPY (SELECT * FROM t) TO 'out.parquet' (FORMAT PARQUET)")
con.sql("EXPORT DATABASE 'dir' (FORMAT PARQUET)")

🧠 Analytics SQL

-- full window functions, CTEs, QUALIFY
SELECT *, row_number() OVER (PARTITION BY dept
         ORDER BY amt DESC) rn
FROM t QUALIFY rn = 1;             -- dedup

SELECT list(name), sum(amt) FROM t GROUP BY dept;
SELECT * FROM t SAMPLE 10%;
PIVOT t ON month USING sum(amt);
FROM t SELECT dept, sum(amt);      -- FROM-first syntax

🎯 Why DuckDB

  • Embedded (a library, no server) + columnar/vectorized → fast analytics locally.
  • Query Parquet/CSV/JSON & DataFrames in place — often replaces a pandas ETL step.
  • Great for local dev, tests, notebooks, and lightweight pipelines.
  • Full SQL: window functions, CTEs, PIVOT, list/struct types.
  • Pairs with dbt-duckdb for local transformations.

⚡ Tips

  • Push work into one SQL statement — the optimizer + parallelism beat row-by-row Python.
  • Use read_parquet globs; DuckDB prunes columns & row groups automatically.
  • Set a memory limit: SET memory_limit='4GB'; spills to disk when needed.
  • Round-trip to pandas/Polars/Arrow with .df()/.pl()/.arrow() (zero-copy).
