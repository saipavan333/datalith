Setup & basics

⚙️ Config & start

git config --global user.name  "Ada"
git config --global user.email "ada@x.com"
git config --global init.defaultBranch main

git init                # new repo
git clone URL           # copy remote
git clone URL dir

💾 Stage & commit

git status              # what changed
git add file.py         # stage one
git add -A              # stage everything
git add -p              # stage hunks interactively
git commit -m "feat: add loader"
git commit -am "msg"     # add tracked + commit
git commit --amend      # fix last commit

🔬 Inspect history

git log --oneline --graph --all
git log -p file         # commits + diffs for a file
git show HEAD           # last commit detail
git diff                # unstaged changes
git diff --staged       # staged changes
git blame file          # who changed each line
Branching & merging

🌿 Branches

git branch                 # list
git switch -c feature      # create + switch (new)
git checkout -b feature    # create + switch (classic)
git switch main
git branch -d feature      # delete (merged)
git branch -M main         # rename current -> main

🔀 Merge vs rebase

git merge feature       # merge into current branch
# linear history: replay commits on top
git switch feature
git rebase main
# interactive: squash/reorder/edit
git rebase -i HEAD~3

Merge keeps history as-is (merge commit). Rebase rewrites for a linear log — don't rebase shared/pushed branches.

☁️ Remotes

git remote -v
git remote add origin URL
git remote set-url origin URL   # change remote
git push -u origin main         # push + track
git push       git pull         # pull = fetch + merge
git fetch       # download, don't merge
git pull --rebase   # linear pull
Fixing things

📦 Stash

git stash              # shelve dirty changes
git stash -u           # include untracked
git stash list
git stash pop          # reapply + drop
git stash apply        # reapply, keep
git stash drop

Use before switching branches with uncommitted work.

↩️ Undo (safe → nuclear)

git restore file          # discard unstaged edits
git restore --staged file # unstage (keep edits)
git revert HEAD           # new commit that undoes (safe)
git reset --soft HEAD~1   # undo commit, keep staged
git reset HEAD~1          # undo commit, keep edits
git reset --hard HEAD~1   # discard commit + edits (!)

revert = safe on shared branches. reset --hard destroys work — be sure.

🍒 Cherry-pick & tags

git cherry-pick a1b2c3    # apply one commit here
git tag v1.0.0            # lightweight tag
git tag -a v1.0.0 -m "release"
git push origin --tags

⚔️ Merge conflicts

# after a conflicting merge/rebase:
git status                 # see conflicted files
# edit files, remove markers:
<<<<<<< HEAD  … ======= … >>>>>>> feature
git add resolved_file
git commit          # (merge)
git rebase --continue   # (rebase)
git merge --abort   /   git rebase --abort

🚫 .gitignore & recovery

# .gitignore
__pycache__/
*.pyc
.env
.venv/
data/*.csv

git rm --cached secret.env  # stop tracking, keep file
git reflog                  # find "lost" commits
git reset --hard HEAD@{2}   # jump back to a reflog state

Committed a secret? Rotate it — history is public even after deletion.

🔁 Daily flow

git switch -c feat/x
# …edit…
git add -A && git commit -m "feat: x"
git push -u origin feat/x
# open PR, review, merge
git switch main && git pull
