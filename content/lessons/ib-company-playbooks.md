# Company playbooks: Google, Meta, Amazon, Goldman, JPMorgan

Each employer has a recognizable loop and emphasis. Tailor your prep to the company in front of you.

## Google

- **Process:** recruiter → 1-2 technical phone screens → onsite of **4-5 rounds (45-60 min)**.
- **Tests:** coding/DSA rigor, SQL, data modeling, system/pipeline design, behavioral ("Googleyness & leadership").
- **Edge:** clean, communicative problem-solving; **BigQuery/GCP** fluency; strong fundamentals.
- **Sample asks:** window-function SQL; design a data warehouse / batch pipeline; graph/tree coding; "tell me about an ambiguous project".
- **Prep focus:** coding cleanliness + design breadth + GCP vocabulary.

## Meta

- **Process:** recruiter → technical screen → onsite of **4 technical + 1 behavioral**, often a single day.
- **Tests:** **data modeling weighted most** (via a **product-sense** prompt), SQL, Python.
- **Edge:** translate a product question into a model (grain, dims, SCD); show measurable impact; "what did *you* do".
- **Sample asks:** "Model engagement for a feed product"; metric SQL (DAU, retention); Python data manipulation.
- **Prep focus:** product-sense → data modeling, plus crisp SQL.

## Amazon

- **Process:** recruiter → online assessment / screen → the **loop** (with a **Bar Raiser**).
- **Tests:** **Leadership Principles in every round, including coding** — bring **12-15 STAR stories**; SQL, DSA, data modeling, design.
- **Edge:** every technical answer is also an LP signal (Ownership, Dive Deep, Deliver Results).
- **Sample asks:** SQL + LP combo; "design an ingestion pipeline"; "tell me about a time you took ownership of a failure".
- **Prep focus:** LP stories tagged to many principles + solid SQL/DSA/design.

## Goldman Sachs

- **Process:** **CoderPad** technical + SQL + distributed-systems + data modeling + values behavioral.
- **Tests:** algorithms (arrays, trees, recursion, simple graphs); **medium-hard SQL** (multi-join, windows, CTEs, optimization); **distributed systems (Spark/Hadoop)**; ETL architecture & data modeling; behavioral on **Partnership, Client Service, Integrity, Excellence**.
- **Edge:** clarity under time pressure; reason about large-scale data workflows; values fit.
- **Sample asks:** CoderPad tree/recursion; "optimize this slow query"; Spark shuffle/skew; "a time you acted with integrity".
- **Prep focus:** CoderPad DSA + hard SQL + Spark internals + values STAR.

## JPMorgan Chase

- **Process:** recruiter → technical (often **Python**; core CS, sometimes **Java/concurrency**, big-data concepts) → **4-5 short onsite rounds (30-35 min)** + case study + behavioral.
- **Tests:** medium DSA in Python (arrays/strings/trees/graphs); SQL (joins, windows, aggregations, subqueries); case studies; **motivation** ("why JPMorgan / why finance").
- **Edge:** breadth across CS + data + cloud; clear communication; genuine interest in finance.
- **Sample asks:** Python array/string problem; window-function SQL; "design a pipeline for transaction data"; "why JPMorgan?".
- **Prep focus:** Python/CS breadth + SQL + a crisp motivation story.

## Tailoring cheat sheet

| Company | Drill most |
|---|---|
| Google | coding rigor + system design + GCP/BigQuery |
| Meta | **data modeling** (product-sense) + SQL + Python |
| Amazon | **12-15 LP stories** + SQL/DSA/design (Bar Raiser) |
| Goldman | CoderPad + hard SQL + Spark/distributed + **values** |
| JPMorgan | Python/CS breadth + SQL + case study + **"why finance"** |

**Rule:** match your strongest prep block to the company's known emphasis, and bring company-specific behavioral stories.
